# ML Fraud Detection

Сервис для автоматического обнаружения мошеннических транзакций в режиме батчевого скоринга. Обрабатывает CSV-файлы из указанной директории с использованием предобученной XGBoost модели.

## Архитектура решения

```text
📦 root/
 ├── 📂 app/
 │ └── 📜 app.py # Ядро сервиса с обработчиком файлов
 ├── 📂 src/
 │ ├── 📜 preprocessing.py # Пайплайн обработки данных
 │ └── 📜 scorer.py # Модуль прогнозирования
 ├── 📂 trained_entities/
 │ ├── 📜 feature_columns.pkl # Файл со списком фичей
 │ ├── 📜 catboost_encoder.pkl # Предобученный на текстовых переменных CatBoost Encoder
 │ ├── 📜 onehot_encoder.pkl # Предобученный на категориальных переменных (идентификатора товара и пола) OneHotEncoder
 │ └── 📜 model.pkl # Предобученная модель XGBoost Classifier
 ├── 📂 input/ # Директория для загрузки файлов на скоринг
 ├── 📂 output/ # Директория с результатами скоринга, json-файла с топ-5 фичей, графика плотности распределения скоров
 ├── 📜 requirements.txt # Список внешних зависимостей и их версий, необходимых для работы сервиса
 ├── 📜 .gitignore
 ├── 📜 Dockerfile
 └── 📜 README.md
```
## Ключевые особенности

### Многоуровневое логирование
- Регистрация всех событий в файл `/app/logs/service.log`
- Консольный вывод для мониторинга в реальном времени
- Три уровня детализации:
  - `INFO`: Основные этапы обработки
  - `DEBUG`: Детали преобразования данных
  - `ERROR`: Критические сбои с трейсбэками

### Пайплайн обработки данных (`preprocessing.py`)
1. Перевод колонки с датой и временем в Unix-формат
2. Создание колонки `full_name` из отдельных признаков `name_1` и `name_2`
3. Кодирование переменных `cat_id` и `gender` через OneHotEncoder
4. Кодирование текстовых переменных через CatBoostEncoder

### Модельный слой (`scorer.py`)
- Порог классификации: 0.97
- Автоматическая загрузка модели при инициализации
- Батчевая обработка через `predict_proba`

## Запуск сервиса
1. Скопируйте репозиторий на локальную машину
2. Соберите Docker-образ

```bash
docker build -t mlops_hw1 .
```
3. Запустите контейнер с монтированием томов
```bash
docker run -it --rm -v ./input:/app/input \
                    -v ./output:/app/output \
                    mlops_hw1
```
4. После запуска сервиса (появления в логах сообщения: `__main__ - INFO - File observer started`) можно приступать к скорингу данных:
 - Разместите файл формата test.csv из соревнования [Teta ML 1 2025](https://www.kaggle.com/competitions/teta-ml-1-2025) в директории `./input`
 - Подождите выполнения препроцессинга и скоринга датасета (в логах будет указано название сформированного файла)
 - Подождите выполнения выгрузки json-файла с топ-5 фичей и графика плотности распределения скоров
 - Полученный результат будет выгружен в директорию `./output`
